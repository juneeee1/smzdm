# 北京81销售体系内容分发系统

为销售人员提供专属推广物料的快速生成与分发系统，实现**推广来源可追踪、物料可下载**。

## 功能简介

系统支持「个人销售员」和「公司」两种身份，登录后进入对应后台。

### 个人（销售员）
- **注册**：姓名、手机号、密码、所属门店、工号
- **门店信息区**：展示所属门店及个人信息
- **渠道二维码申领**：展示专属二维码 + 个人渠道号，支持下载
- **营销内容申领**：Canvas 合成推广海报（底图 + 二维码 + 渠道号 + 个人/门店信息），支持下载

### 公司
- **注册**：公司名称、统一社会信用代码、联系人姓名、联系电话、密码、公司地址
- **渠道号展示**：展示公司渠道号及基础信息（公司名称、注册时间）

## 核心规则

| 项目 | 规则 | 示例 |
| --- | --- | --- |
| 个人渠道号 | `P` + 时间戳后6位 + 随机3位 | `P123456ABC` |
| 公司渠道号 | `C` + 时间戳后6位 | `C123456` |
| 二维码内容 | `https://www.smzdm.com?channel={渠道号}` | — |

## 技术栈

- **Vue 3** + **TypeScript**
- **Vite** 构建（`base: './'`，确保静态资源在本地文件系统可加载）
- **Vue Router**（Hash 路由 + 登录态守卫）
- **Pinia** 状态管理
- **qrcode** 二维码生成
- **localStorage** 本地存储（数据结构预留 `id` 字段，便于未来对接后端）

## 目录结构

```
src/
├── components/
│   ├── auth/          # 登录、个人/公司注册表单
│   └── dashboard/     # 门店信息、二维码面板、海报面板等
├── views/             # LoginView、DashboardView 页面
├── router/            # 路由与登录守卫
├── stores/            # Pinia 用户状态
├── types/             # 类型定义（PersonalUser / CompanyUser）
├── utils/             # 渠道号生成、初始化数据、storage 封装
└── styles/            # 全局样式
```

## 本地开发

> 需要 Node.js 18+（开发环境使用 v22）。

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build

# 本地预览构建产物
npm run preview
```

## 部署

项目已配置 `vercel.json`，可直接通过 [Vercel](https://vercel.com/) 部署。
